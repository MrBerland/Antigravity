Manufacturing Module System Administration Version 2.1

Stoppage Rule Configuration

The semantic foundation of the manufacturing intelligence module. Configuration defines the ruleset by which operators describe reality, creating the ontology that makes mathematical analysis possible.

1. Defining the Semantic Model

If fifty operators are allowed to type whatever they want into a text box, the plant generates zero analytically viable data. "Machine broke," "Blt snapping," "Belt snapped," and "conveyer stop" are all fundamentally the same root cause, but no software can intuitively aggregate them in a Pareto chart.

  • Standardized Terminology: Establishing a uniform lexicon across a manufacturing enterprise ensures that a line in Cape Town uses the identical failure phrasing as a line in Johannesburg.
  • Bounded Context: By forcing selections from a dropdown matrix, the system bounds the context. Operators do not need to diagnose complex electrical faults; they only need to record the symptom and the general category for maintenance down the line.
The "Too Broad / Too Narrow" Rule

A poorly configured matrix destroys OEE tracking. If the Level 3 reasons are too broad (e.g., "Maintenance Check"), no actionable CI target is created. If they are too narrow (e.g., "Left Sieve Arm Set-Screw Loosened"), operators will spend 5 hours searching a 10,000-item list for the right code. The Augos best practice typically dictates 5 to 7 Level-1 Categories, with max 10 to 15 sub-items deep.

2. Structural Matrix Rules

The Augos Admin portal allows plant engineers to dynamically build and nest the stoppage matrix, attributing nodes directly to OEE classification buckets.

Hierarchical Action Data Integrity Consequence
Planned vs Unplanned Tagging At the highest foundational setup, every reason node must classify as either an "Availability Loss" (Unplanned) or an OEE exclusion event (Planned Stop / Shift End).
Nesting Level 2 inside Level 1 "Flour Bin Full" (Level 3) must be strictly nested inside "Downstream Logistics" (Level 2). It cannot sit redundantly inside "Mechanical Failure."
Global vs Local Deployment A stoppage tree can be built universally for the entire factory, or localized down to a specific Point ID (e.g., the Filler machine has completely different failure modes than the Palletizer).

3. Working UI Component View

The admin configuration panel visualizes the stoppage matrix as an editable recursive tree.
